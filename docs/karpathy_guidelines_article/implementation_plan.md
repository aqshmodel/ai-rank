# 10,000文字執筆計画: Karpathy Guidelinesの適用

このドキュメントは、Andrej Karpathy氏のコーディングガイドラインに関する約10,000文字の記事をThe AI Rankのブログとして執筆・最適化するための実装計画です。

## 目的
`karpathy_guidelines_blog_draft.md.resolved` の内容を元にし、The AI Rank特有の「Human-Like Style（AI臭の排除・共感と主観）」に基づいた圧倒的な熱量を持つ10,000文字の超長編記事を生成する。

## 実装ステップ

1. **フェーズ1: 骨組みと日常の絶望層の構築（約3,000〜4,000文字）**
   - 導入部にて「AIにコードを壊された絶望感」の泥臭い体験談を描写。
   - Karpathy原則の「1. Think Before Coding」を中心に、人間としての前提を言語化する苦労を盛り込む。

2. **フェーズ2: 具体的事例と対比の追加（約6,000〜7,000文字）**
   - 「2. Simplicity First」「3. Surgical Changes」の解説。
   - 不要な抽象化と外科的変更の概念を、実際の開発現場（例えば深夜のコードレビュー）の温度感とともに詳述する。

3. **フェーズ3: 組織論と伴走の必要性（約8,500〜9,000文字）**
   - 「4. Goal-Driven Execution」を通し、チームとしてAIとどう向き合うのか（温度差の問題）を展開。
   - 単なるツール導入（プロンプト等）では解決しない企業のリアルな痛みと、Aqshの伴走コンサルの必要性へ繋げる。

4. **フェーズ4: 最終調整・The Squeeze（10,000文字到達へ）**
   - 感情的な後書きや表記揺れの意図的な追加。
   - Pythonスクリプトによる文字数カウントおよび微調整。

5. **フェーズ5: アセット作成とSEO**
   - カバー画像の生成（1200x630, WebP）。
   - `posts/karpathy-ai-coding-guidelines.md` として保存し、メタデータを設定。
