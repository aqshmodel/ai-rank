# The AI Rank 執筆・メディアガイドライン

## 1. コア・ライティングスタイル (Human-Like Style)
生成AIの普及により、世の中は無機質で当たり障りのないSEO記事で溢れています。The AI Rankでは、これを逆手に取り**「圧倒的に人間らしく、主観的な体温のある文章」**を標準とします。

### 禁止事項（AI臭の排除）
- 記事内で「」（直角括弧）を多用しないこと。（例: 「AI導入のメリット」とは、「コスト削減」と「効率化」です。 ← NG）
- 冒頭で「本記事では〜について解説します。ぜひ最後までお読みください」といった定型的な挨拶をしない。
- 当たり障りのない一般論（第三者目線）ではなく、常に「私たち（Aqsh）の現場の経験上〜」という一次情報の視点で語ること。

## 2. 記事構成の型パターン
ユーザーインサイトに基づき、すべての記事は以下の流れで構成します。
1. **Pain（共感・問題提起）**: 「〜で失敗していませんか？」「〜という絶望的な状況」
2. **Analysis（常識の否定・分析）**: 「なぜツールを入れるだけではダメなのか」
3. **Solution（独自の解決策）**: The AI Rankによる可視化と、個社専用エージェント構築という現場に寄り添うアプローチ。
4. **CTA（行動喚起）**: 無料診断、または岩手法人向けLPへの誘導。

## 3. アセット（画像）管理ルール
### ディレクトリ構造
- カバー画像や記事中の挿絵はすべて **`/assets/articles/[記事のslugファイル名]/`** 内に格納する。
- 例: `local-ai-failure.md` の画像は `/assets/articles/local-ai-failure/cover.webp` 等にする。

### カバー画像フォーマット
- 推奨サイズ: 1200 x 630 px
- 形式: **WebP推奨**（または高圧縮PNG）
- ファイル名: 必ず `cover.webp` や `cover.png` とする。
