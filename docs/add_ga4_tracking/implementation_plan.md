# GA4 トラッキング実装およびフロントエンド読み込み修正計画（第3者レビュー反映版）

## 概要

THE AI RANK いわてのプロジェクトにて、Googleアナリティクスでアクセス解析を強化し、コンバージョンとユーザー体験を正確に把握するためのカスタムイベントを実装します。
同時に、現在のHTMLテンプレート内でリンク切れとなっている旧スクリプト参照問題を解決し、`js/main.js`等への結線を行います。

**【レビュアーからの指摘による重要修正方針】**
1. **SPA特有の「完読」の再定義**: 診断ページはスクロールではなく「画面の切り替え」で進むため、単純なスクロール率では完了や離脱が測れません。ページタイプに応じた適切な追跡が必要です。
2. **ESモジュール化の副作用**: `<script src="script.js">` を無邪気に `<script type="module" src="/js/main.js">` に置換すると、グローバルスコープが閉じるため、インラインの onclick イベント等が参照エラー（`関数 is not defined`）になるリスクがあります。これを防ぐためのエクスポートと確認が必要です。
3. **CTAと外部遷移の区別**: 内部回遊と外部（コーポレートサイトの問い合わせなど）への遷移を明確にパラメーターで区別する必要があります。

## 実装内容（アップデート版）

### 1. トラッキングラッパー関数の用意
`js/main.js` 内に、GA4へイベントを送信するためのヘルパー関数を実装し、エラー防止のため `window.gtag` の存在を確認してから安全に発火させます。

### 2. ページタイプ別の「完読率」と「SPAファネル」計測仕様
- **記事・用語集など（通常ページ）**: 
  - Intersection Observer 等を利用し、各ページの25%, 50%, 75%, 100%スクロール到達時に `scroll_depth` イベントを送信します。
- **診断ページ（SPA挙動）**: 
  - スクロール深度ではなく、**各ステップの通過到達率（ファネル）**として計測します。
  - 名前入力時: `diagnosis_start`
  - 各設問回答時: `diagnosis_step` (パラメーター: `step_index: N`)
  - 診断完了（判定）時: `diagnosis_complete` (パラメーター: `rank_level: X`, `rank_name: 'VIBE CODER'`) 
  - ※これにより、ユーザーがQ3で離脱したのか、Q10まで行ったのかが明確に分析可能になります。

### 3. CTAクリックイベントの計測（粒度向上）
重要なCTAボタンにイベントリスナーを追加しますが、遷移先が内部か外部かを計測パラメーターで明確にします。
- **イベント名**: `click_cta`
- **パラメーター**: 
  - `cta_label`: 「診断を始める」「お問い合わせ」など
  - `cta_type`: `internal`（自サイト遷移）か、`external`（Aqsh社サイトなど別ドメイン遷移）か
  - `location`: `hero`（メイン）、`header`、`footer`など

### 4. シェア・ダウンロードのアクション計測
ユーザーのバイラル行動を計測するため、証明証生成やSNSへのシェアボタンのイベント送信を担保します。
- `share`イベント（パラメーター: `method: 'X'`, `method: 'LinkedIn'`)
- カスタムイベント: `download_certificate`

### 5. [WARNING] フロントエンドスクリプトの参照修正とスコープ対応
全HTMLテンプレートの `<script src="script.js?v=..."></script>` を `<script type="module" src="/js/main.js"></script>` へ置換します。
**【不可欠な手順】**：モジュール化に伴い、`onload`, `onclick`, インラインスクリプトから呼ばれているグローバル関数（例: `window.openModal` など）がエラーにならないよう、`js/main.js` 内で意図的に `window.openModal = function...` 等のグローバル露出を確実に行います。
