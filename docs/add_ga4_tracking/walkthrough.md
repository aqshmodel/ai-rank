# GA4トラッキングへのアップデート完了報告

AI RANK いわてのフロントエンドに対し、Google Analytics (GA4) のカスタムイベントトラッキング実装および、リンク切れの発生していた古いスクリプトのモジュール移行を完了しました。

## 概要

本対応により、ユーザーのランディングから診断完了・シェアといった「成果（CV）」に直結する行動と、完読率などのエンゲージメント指標を正確にGA4側で計測できるようになりました。また、第三者視点のレビューにて特定した「SPA特有の考慮漏れ」や「モジュール化による副作用リスク」についても確実に対応済みです。

## 実施した変更内容

### 1. `js/main.js`におけるトラッキングの実装
以下のような主要アクションに対し、GA4へのイベント送信（または `dataLayer.push`）を行うラッパー関数 `window.trackGa4Event` を実装しました。

- **完読率の計測**
  -  ページのスクロール位置を監視し、25%、50%、75%、100% を通過したタイミングで `scroll_depth` イベントを送信しています。
- **診断プロセスのファネル解析（SPA対応）**
  - **診断開始**: `diagnosis_start` を送信
  - **進行（各設問への到達）**: `diagnosis_step` ( `step_index: N` ) を送信
  - **診断完了**: `diagnosis_complete` ( `rank_level: X`, `rank_name: 'xxx'` ) を送信
  ※ これにより、どこでユーザーが離脱したかをステップごとに詳細に把握できます。
- **コンバージョン・CTA要素のクリック**
  - `click_cta` イベントを追加。「診断を始める」や「お問い合わせ」などのボタンクリックを捕捉します。
  - 同時に、ドメイン（自サイトか `aqsh.co.jp` へ向かう外部送客か）を判断し `cta_type: 'internal'` / `'external'` 等のパラメーターを付与しました。
- **バイラルと拡散行動**
  - 証明書画像の生成 (`download_certificate`)
  - SNSシェア (`share` [ `method: 'X'` / `method: 'LinkedIn'` ])

### 2. リンク切れ旧・スクリプトの修正とスコープ対応
HTMLテンプレート側で404エラーを引き起こしていた `<script src="script.js">` などを、最新の `<script type="module" src="/js/main.js">` へ張り替えました（`index.html` および `diagnosis.html`）。

**副作用対策**:
モジュール化に伴い発生しうる「HTMLインライン（`onclick="openModal()"`等）からのアクセスエラー」を防ぐため、`js/main.js` の末尾において、必要な関数を意図的にウィンドウのグローバルオブジェクト(`window.openModal` / `window.closeModal`)へとエクスポートする処理を入れています。

## 検証結果（動作確認）

各ページのソースコードが正しくビルドされること、GA4の `gtag` または `dataLayer` へのpushリクエストが発火するJavaScriptの構造にエラーがないことを目視および静的解析で確認しました。

本番やステージング環境にてデベロッパーツール（`window.dataLayer` の動きや、NetworkタブでのGAへの通信）をご確認いただければ、トラッキングが実行されていることが観察できます。
