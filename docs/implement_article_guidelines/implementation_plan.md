# 次のステップ：SEOテクニカル改善および自動化の提案

数十記事規模へとスケールさせ、The AI Rankのオーガニック（検索）流入を本気で獲得していくために、現在のプロジェクトを調査した結果、**最優先で実装すべき3つのテクニカルSEO改善**が浮上しました。

## User Review Required

> [!IMPORTANT]
> **構造化データとサイトマップの自動化について**
> 以下に提案する3つの機能は、運用効率とGoogleからの評価（E-E-A-Tのシステム的な伝達）を飛躍的に高めるものです。手作業で構築するには手間がかかりすぎるため、`server.js` に自動処理を組み込みたいと考えています。
> 内容をご確認いただき、問題なければ実装へと進めさせてください。

## Proposed Changes (追加提案事項)

### 1. 動的 `sitemap.xml` の生成エンドポイント追加
**理由**: 現在サイトマップが存在せず、Googleが新しい記事（何十記事）を見つけるのに時間がかかります。
**実装内容**:
- `server.js` に `app.get('/sitemap.xml')` のルートを追加。
- アクセス時に自動で `posts/` フォルダ内の `.md` を走査し、`index.html` や `iwate.html` を含めた完全なXMLを動的生成・出力します。記事追加のたびに何か手動で実行する必要はなくなります。

### 2. JSON-LD（構造化データ）の自動生成と埋め込み
**理由**: 現在、記事のページには人間向けの「著者プロフィール」は見えていますが、Googleのボット（機械）に「この人は専門家である」と伝えるための**メタデータ（schema.org）**が欠落しています。
**実装内容**:
- `server.js` のブログ生成関数内において、著者の情報（`authorMaster`）と記事情報（タイトル、日時など）をマージし、`<script type="application/ld+json">` の形式で出力する機能を追加します。
- これにより、Googleが「この記事はAI専門家の塚田氏が書いたものだ」とシステムレベルで正確に解釈できるようになります（Prismaでも採用されている重要なE-E-A-T戦略です）。

### 3. パンくずリスト (Breadcrumbs) の導入
**理由**: ユーザーが検索から直接「記事ページ」に落ちてきた場合、The AI RankのTOPページやブログ一覧へ遷移せず離脱してしまうリスクが高いため。
**実装内容**:
- `views/article-template.html` の記事上部に `THE AI RANK > ブログ・事例 > 記事タイトル` というパンくずリスト（階層リンク）を設置。
- 一緒にパンくずリスト用の構造化データも付与し、SEOでの表示結果をリッチ化します。
