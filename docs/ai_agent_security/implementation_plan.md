## 提案する記事構成案（プロット）

**タイトル案（13文字以内）**：AIエージェントと死の罠
**ターゲットキーワード**：`AIエージェント とは` `Claude Code 使い方` `Cursor セキュリティ` 
**ペルソナ**：AIツールを導入した（しようとしている）企業の経営者・DX推進担当者

### 1. Pain（問題提起と共感の導入） - 約2,000字
*   **現場の惨状**：「全社員にAIプログラミングツール（CursorやClaude Code）を配って『これで開発速度が10倍になる』と喜んでいる経営者たちへ」
*   **自律型AIの脅威**：これまでのChatGPTとは違い、AIが「自分でファイルを書き換え」「コマンドを叩き」「外部と通信する」ことの本当の恐ろしさ。
*   **悲劇のシナリオ**：意図しない `rm -rf` による環境破壊、.env ファイルの漏洩、悪意あるリポジトリの clone によるマルウェア感染のリアルな描写。

### 2. Analysis（常識の否定・分析） - 約3,000字
*   **「ルールで縛る」の限界**：現場のITリテラシーへの過信が引き起こすシャドーAIとパスワード直書きの恐怖。
*   **OWASPによる新時代のサイバーリスク**：プロンプトインジェクションは遠い世界の話ではない。Antigravityのようなツールですら、プロンプトに細工されるとサンドボックスを抜け出されるリスクがあること。
*   **ツール別・油断しやすい落とし穴**：
    *   `Cursor`：Privacy Modeを切った状態での機密コード漏洩リスク。
    *   `Claude Code`：便利なMCP（Model Context Protocol）連携が引き起こす外部サービスへの不正アクセス連鎖。
    *   `Copilot / Codex`：古い脆弱性のあるコードをそのまま学習・提案してしまうリスク。

### 3. Solution（現場目線の本質的アプローチと”18の絶対防壁”） - 約5,000字
*   **自己責任論を捨てる**：ガチガチのセキュリティで縛ると誰も使わなくなる。実用性と安全性を両立させるための「誰でもできる」具体的な防波堤。
*   **【環境・権限のコントロール】（元ネタ対策1, 5, 6, 12等より）**
    *   **「ホームディレクトリの悲劇」を防ぐ**：作業用空フォルダ (`mkdir`) を起点としたサンドボックス化（WSL2やDevContainerの活用）。
    *   **悪魔のコマンド**：`--dangerously-skip-permissions` を使うエンジニアは現場から追放すべき理由。askモード（実行前確認）の絶対死守。
    *   **MCPの恐怖**：便利さの裏にある外部サービスへの不正アクセス連鎖。不要なMCPサーバー連携の遮断と `/permissions` による定期確認。
*   **【データの徹底保護】（元ネタ対策2, 3, 10, 11, 13等より）**
    *   **「.env絶対置かないマン」の徹底**：APIキー漏洩の王道パターンを潰す。設定はチャットに直書きせず環境変数で渡す運用。
    *   **API上限と学習オフ**：Claudeの学習データ使用オフ（Pro版でも注意）と、Spending Limit（利用上限）設定による金銭的ガード。
    *   `/clear` コマンドによる不要なコンテキスト（会話履歴）の定期リセットの重要性。
*   **【検証と実行のプロセス】（元ネタ対策4, 7, 8, 14, 15等より）**
    *   **「CLAUDE.md」を使った最強の盾**：プロジェクト直下に自己制約ルール（猫をかぶらせる条件）を明記し、AI自身にルールを遵守・監視させる。
    *   **Plan Modeの癖をつける**：実行前に「何をするか」をAIに説明させるひと手間で9割の事故は防げる。
    *   **インジェクション対策**：安易な外部URLの貼り付けと、悪意あるリポジトリの `clone` がもたらす最悪のシナリオ回避。
    *   **人間による最終防衛線**：`git commit` と `WORKLOG.md` の併用。「AIが書いたコードは動いても信じるな」という目視（git diff）の鉄則。

### 4. CTA（The AI Rank・Aqshへの結実） - 約1,000字
*   **組織のAIリテラシー総点検**：セキュリティはどんなにツールを設定しても「人間の頭の中の意識」に依存する。だからこそ「THE AI RANK いわて」で全社員のレベルを可視化しなければならない。
*   **現場に伴走するコンサルティングの意義**：丸投げのツール契約をやめ、自社のワークフローに合わせた「安全な専用AIエージェント環境（Antigravity等を用いたカスタマイズ）」を構築するためのAqshへの問い合わせ誘導。
