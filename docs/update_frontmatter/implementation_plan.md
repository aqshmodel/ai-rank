# ブログ記事フロントマター改善計画

現在、`posts/` ディレクトリ配下にある7つの記事のフロントマター（メタデータ）について、ガイドライン（タイトル13文字以内）から逸脱している問題や、表記揺れ、カテゴリの欠落などが散見されます。
本計画ではこれらを一斉に改修し、LPとしてのCTR（クリック率）やSEO効果を最大化するための見直しを提案します。

## 課題の背景と方針

1. **タイトルの文字数超過**: ほとんどの記事が13文字を大幅に超過しており、「圧倒的に人間らしく、主観的な体温のある」LP的キャッチコピーとして最適化されていませんでした。
2. **メタデータの欠落**: 一部の記事に `category` や `coverImage` が設定されていません。
3. **フォーマットの揺れ**: `date` フィールドの形式（ハイフンとドットの混在）や画像拡張子（`.png` と `.webp` の混在）を統一します。

## ユーザレビュー確認事項

以下の新しいタイトル（13文字以内）および追加カテゴリの素案について、方針が問題ないかご確認ください。

> [!IMPORTANT]
> **懸念点・確認事項**
> もし既存の長いタイトルをOGP（SNSシェア時の説明文等）に残したい場合は、新規に `ogTitle` といったフィールドを追加することも可能です。今回は現行のテンプレート（`article-template.html`）が `{{title}}` と `{{description}}` の組み合わせで描画されているため、`title` そのものを13文字へと一気に縮めるアプローチを取っています。
